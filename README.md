# Kursova_NVBD_Variant_8
Kursova
Система Data Warehouse та ETL-процеси на базі SSIS і SSAS
Загальний опис проєкту

Даний проєкт реалізує сховище даних (Data Warehouse) з повним циклом ETL-процесів з використанням Microsoft SQL Server Integration Services (SSIS) та аналітичного моделювання за допомогою SQL Server Analysis Services (SSAS).

Метою роботи є проєктування, наповнення та аналіз сховища даних на основі операційної бази даних. Рішення забезпечує структуроване завантаження, трансформацію та багатовимірний аналіз даних із застосуванням інструментів Microsoft Business Intelligence.

Проєкт виконано в межах курсової роботи з дисципліни, пов’язаної з технологіями сховищ даних та аналітики.

Архітектура рішення

Рішення побудовано відповідно до класичної архітектури Data Warehouse:

Операційна база даних (OLTP):
InventoryDB — джерело транзакційних даних.

Сховище даних (OLAP):
GoodsDW — аналітична база даних.

ETL-рівень:
Реалізований за допомогою SSIS-пакетів.

Аналітичний рівень:
Реалізований за допомогою SSAS-кубів.

Структура сховища даних

Сховище даних реалізоване за схемою зірки (Star Schema).

Таблиці фактів

FactPayment — містить дані про фінансові операції.

FactInventoryMovement — містить дані про рух товарів.

Таблиці вимірів

DimDate

DimCustomer

DimSupplier

DimProduct

DimOperationType

Таблиці фактів пов’язані з таблицями вимірів через сурогатні ключі.

ETL-процеси (SSIS)
Створення проєкту

Встановлено SQL Server Data Tools (SSDT).

Створено проєкт типу Integration Services.

Налаштовано OLE DB-з’єднання з базами InventoryDB та GoodsDW.

Витягування даних (Extract)

Для витягування даних використано компоненти OLE DB Source з таких таблиць:

dbo.Payment

dbo.Operation

dbo.OperationLine

довідникові таблиці для Lookup-операцій.

Трансформація даних (Transform)

У процесі трансформації реалізовано такі етапи:

Data Conversion

Приведення типів даних до цільових форматів.

Стандартизація дат і числових значень.

Derived Column

Нормалізація текстових полів.

Обробка NULL-значень.

Формування похідних атрибутів.

Lookup

Отримання ключів дат із таблиці DimDate.

Визначення типів операцій через довідники.

Збагачення фактів даними з вимірів.

Обробка помилок

Використання виходів No Match для запобігання аварійному завершенню ETL.

Некоректні записи не порушують виконання пакетів.

Завантаження даних (Load)

Завантаження даних у таблиці фактів за допомогою OLE DB Destination.

Використано режим Fast Load.

Забезпечено коректну роботу з ідентифікаторами та ключами.

Аналітична модель (SSAS)

На основі сховища даних створено OLAP-куб.
